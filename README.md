# CS 250 Portfolio – Sprint Review & Retrospective

## Reflection

In this project, I learned how to understand user needs and turn them into working features using Agile practices. I interpret user needs by breaking them into smaller tasks and focusing on what the user wants. Creating user stories helps me clearly understand the user’s goal and expected result.

When I develop programs, I prefer using Agile methods because they allow flexibility and continuous improvement. I like working in small steps, testing regularly, and improving based on feedback. In the future, I want to continue using daily Scrum meetings, sprint planning, and continuous testing.

Being a good team member means communicating clearly, being responsible, and helping others. I also learned that teamwork is very important in Agile because everyone depends on each other. A good team member supports the team and contributes positively.
